# Implementation Plan

## Tasks

- [x] 1. スプラッシュ画面の基盤セットアップ
- [x] 1.1 フィーチャーディレクトリ構造の作成
  - splashフィーチャー用のディレクトリ（pages, providers, models）を作成する
  - 初期化結果を表す型（成功、タイムアウト、ネットワークエラー）を定義する
  - sealed classパターンで型安全な結果型を実装する
  - _Requirements: 3.1, 3.2_

- [x] 1.2 (P) ルーティング設定の更新
  - アプリのエントリーポイントをスプラッシュ画面に変更する
  - スプラッシュ画面からホーム画面への遷移ルートを追加する
  - go_routerの設定でスプラッシュ画面を初期表示に設定する
  - _Requirements: 2.1_

- [x] 2. 初期化ロジックの実装
- [x] 2.1 ホームデータ取得プロバイダーの作成
  - タイムラインデータを取得するプロバイダーを作成する
  - APIクライアントを使用してデータをフェッチする
  - エラー時は例外をスローしてスプラッシュ側でハンドリングできるようにする
  - 本番APIが未実装の場合はモックデータで代替する
  - _Requirements: 2.1_

- [x] 2.2 スプラッシュ初期化プロバイダーの作成
  - 最低表示時間（1.5秒）とデータロードを並行実行するロジックを実装する
  - タイムアウト（10秒）を超えた場合はタイムアウト結果を返却する
  - ネットワークエラーをキャッチしてエラー結果に変換する
  - 成功時、タイムアウト時、エラー時それぞれの結果型を返却する
  - _Requirements: 2.1, 2.2, 2.3, 3.1, 3.2, 3.3_

- [x] 3. スプラッシュ画面UIの実装
- [x] 3.1 基本レイアウトの構築
  - 画面中央にアプリアイコンを配置する
  - アイコンの下部にキャッチコピーを表示する
  - 既存のAppThemeからグラデーション背景を適用する
  - 各種画面サイズに対応したレスポンシブレイアウトを実装する
  - _Requirements: 1.1, 1.2, 1.3, 4.1, 4.2, 4.4_

- [x] 3.2 アニメーションの実装
  - アイコンのフェードイン・スケールアニメーションを実装する
  - キャッチコピーの遅延フェードインアニメーションを実装する
  - 既存のAppTheme定数（durationSlow, curveDefault）を使用する
  - グロー効果でブランド要素を演出する
  - _Requirements: 1.4_

- [x] 3.3 画面遷移の実装
  - 初期化プロバイダーの状態を監視する
  - 初期化完了時にフェードアウトトランジションを実行する
  - 遷移完了後にホーム画面へナビゲートする
  - エラー状態でも遷移を実行してホーム画面でエラー表示を行う
  - _Requirements: 2.1, 2.2, 2.4, 3.1, 3.2_

- [x] 4. 統合とテスト
- [x] 4.1 エンドツーエンド動作確認
  - アプリ起動からスプラッシュ表示、ホーム画面遷移までのフロー確認
  - 正常系（データロード成功）の動作確認
  - タイムアウト時の動作確認（APIを遅延させてテスト）
  - ネットワークエラー時の動作確認（オフラインモードでテスト）
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 2.4, 3.1, 3.2, 3.3, 4.1, 4.2, 4.4_

- [x]* 4.2 (P) ユニットテストの作成
  - 初期化プロバイダーの最低表示時間ロジックをテストする
  - タイムアウト処理のテストを作成する
  - 各結果型の生成とパターンマッチングをテストする
  - _Requirements: 2.3, 3.3_
